# 网站速度优化进阶

## 一、理论框架

### 1.1 网站速度的重要性

网站速度是用户体验的核心要素，直接影响商业表现。研究表明：

**速度与转化的关系：**
- 页面加载时间每增加1秒，转化率下降7%
- 53%的移动用户会放弃加载超过3秒的网页
- 延迟100毫秒可使转化率降低7%
- 亚马逊发现每100ms延迟导致销售额下降1%

**速度与SEO的关系：**
- 2018年Google将移动页面速度纳入排名因素
- 2021年Core Web Vitals成为核心排名信号
- 快速网站获得更好的爬取效率

**速度与用户行为：**
- 加载时间超过3秒，跳出率增加32%
- 快速网站用户浏览页面数多出25%
- 速度影响用户对品牌的信任度

### 1.2 Core Web Vitals核心指标

Google定义的Core Web Vitals是衡量网页体验的关键指标：

**LCP（Largest Contentful Paint）最大内容绘制**
- 定义：最大内容元素渲染完成的时间
- 目标：<2.5秒（良好）
- 需改进：2.5-4秒
- 较差：>4秒
- 影响因素：服务器响应时间、渲染阻塞资源、资源加载速度

**FID（First Input Delay）首次输入延迟**
- 定义：用户首次交互到浏览器响应的时间
- 目标：<100毫秒（良好）
- 需改进：100-300毫秒
- 较差：>300毫秒
- 影响因素：JavaScript执行时间、主线程阻塞

**CLS（Cumulative Layout Shift）累积布局偏移**
- 定义：页面视觉稳定性，元素意外移动的累积分数
- 目标：<0.1（良好）
- 需改进：0.1-0.25
- 较差：>0.25
- 影响因素：无尺寸图片、动态内容插入、字体加载

**2024年新增指标：**
- **INP（Interaction to Next Paint）**：替代FID，衡量交互响应性
- 目标：<200毫秒

### 1.3 网站速度优化模型

**性能优化金字塔：**

```
        ┌──────────────┐
        │   感知优化   │ ← 优先级最高
        ├──────────────┤
        │   代码优化   │
        ├──────────────┤
        │   资源优化   │
        ├──────────────┤
        │   网络优化   │
        ├──────────────┤
        │   服务器优化 │ ← 基础层
        └──────────────┘
```

**关键渲染路径优化：**
1. 构建DOM树（HTML）
2. 构建CSSOM树（CSS）
3. 执行JavaScript
4. 构建渲染树
5. 布局计算
6. 绘制

优化目标：最小化关键渲染路径上的阻塞

### 1.4 性能预算概念

性能预算是为网站设定的速度目标：

**基于时间的预算：**
```
首字节时间（TTFB）：<600ms
首次内容绘制（FCP）：<1.8s
最大内容绘制（LCP）：<2.5s
可交互时间（TTI）：<3.8s
```

**基于大小的预算：**
```
HTML：<50KB
CSS：<100KB
JavaScript：<200KB（压缩后）
图片：<500KB（首屏）
总页面重量：<1MB
```

**基于请求的预算：**
```
总请求数：<50个
关键路径请求：<10个
第三方请求：<15个
```

## 二、实操步骤

### 2.1 性能诊断与分析

**步骤一：建立基准数据**

使用多种工具测试当前性能：

```bash
# 使用Lighthouse CLI
lighthouse https://your-site.com --view

# 使用WebPageTest API
webpagetest test https://your-site.com -k YOUR_API_KEY
```

**关键测试维度：**
- 测试多次取平均值
- 测试移动端和桌面端
- 测试不同网络条件（3G/4G/5G/WiFi）
- 测试不同地理位置

**步骤二：识别性能瓶颈**

常见的性能瓶颈类型：

| 瓶颈类型 | 症状 | 可能原因 |
|---------|------|---------|
| 服务器响应慢 | TTFB高 | 服务器配置差、数据库查询慢 |
| 资源阻塞 | FCP/LCP高 | 同步JS/CSS、字体加载慢 |
| 资源过大 | 加载时间长 | 图片未优化、JS包过大 |
| 主线程阻塞 | FID/INP高 | 长任务、复杂JS |
| 布局不稳定 | CLS高 | 无尺寸图片、字体FOIT |

**步骤三：建立监控体系**

```javascript
// 使用Performance API收集真实用户数据
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log({
      name: entry.name,
      duration: entry.duration,
      startTime: entry.startTime
    });
  }
});
observer.observe({ entryTypes: ['measure', 'navigation', 'resource'] });
```

### 2.2 服务器端优化

**1. 选择高性能主机**

| 主机类型 | TTFB | 适用场景 | 月费 |
|---------|------|---------|------|
| 共享主机 | 500-1000ms | 小流量站点 | $5-20 |
| VPS | 200-500ms | 中等流量 | $20-100 |
| 独立服务器 | 100-300ms | 大流量 | $100-500 |
| 云托管 | 50-200ms | 可扩展需求 | 按需 |

**2. CDN配置优化**

CDN（内容分发网络）是加速的关键：

```
推荐CDN服务商：
- Cloudflare（免费版可用，Pro $20/月）
- Fastly（企业级，按需定价）
- AWS CloudFront（按流量计费）
- BunnyCDN（性价比高，$0.01/GB）
- KeyCDN（$0.04/GB）
```

CDN最佳实践：
- 静态资源全部走CDN
- 设置合理的缓存头
- 启用HTTP/2或HTTP/3
- 配置边缘缓存规则

**3. HTTP缓存策略**

```
静态资源（图片、CSS、JS）：
Cache-Control: public, max-age=31536000, immutable

HTML页面：
Cache-Control: no-cache, ETag: "xxx"

API响应：
Cache-Control: private, max-age=300
```

**4. Gzip/Brotli压缩**

```nginx
# Nginx配置
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml;
gzip_min_length 1024;
gzip_comp_level 6;

# Brotli（更高效）
brotli on;
brotli_types text/plain text/css application/json application/javascript;
brotli_comp_level 4;
```

压缩效果对比：
| 格式 | 原始大小 | Gzip | Brotli |
|------|---------|------|--------|
| HTML | 100KB | 25KB | 20KB |
| CSS | 200KB | 40KB | 32KB |
| JS | 500KB | 150KB | 120KB |

**3. 图片懒加载**

```html
<!-- 原生懒加载 -->
<img src="image.jpg" loading="lazy" alt="">

<!-- 交叉观察器实现 -->
<img data-src="image.jpg" class="lazy" alt="">
<script>
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      observer.unobserve(img);
    }
  });
});
document.querySelectorAll('.lazy').forEach(img => observer.observe(img));
</script>
```

### 2.5 字体优化

**1. 字体加载策略**

```css
/* font-display: swap */
@font-face {
  font-family: 'CustomFont';
  src: url('font.woff2') format('woff2');
  font-display: swap; /* 立即显示后备字体 */
}

/* 字体预加载 */
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
```

**2. 字体子集化**

```bash
# 使用pyftsubset创建子集
pyftsubset font.ttf \
  --text="常用汉字" \
  --output-file=font-subset.woff2 \
  --flavor=woff2

# 使用Unicode范围子集
pyftsubset font.ttf \
  --unicodes="U+0000-00FF,U+4E00-9FFF" \
  --output-file=font-subset.woff2
```

**3. 可变字体**

```css
/* 可变字体减少文件数量 */
@font-face {
  font-family: 'VariableFont';
  src: url('VariableFont.woff2') format('woff2-variations');
  font-weight: 100 900;
  font-style: oblique 0deg 12deg;
}

body {
  font-family: 'VariableFont', sans-serif;
  font-weight: 400; /* 可动态调整 */
}
```
