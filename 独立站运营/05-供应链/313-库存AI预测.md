## 二、库存预测核心维度

### 2.1 需求预测

**预测层级**：
- SKU级别（单个商品）
- 类别级别（品类汇总）
- 仓库级别（地域分布）
- 时间粒度（日/周/月）

**影响因素**：

| 因素类型 | 具体因素 | 数据来源 |
|----------|----------|----------|
| 历史销售 | 销量趋势、季节性 | 订单系统 |
| 营销活动 | 促销、折扣、广告 | 营销系统 |
| 外部因素 | 节日、天气、事件 | 第三方API |
| 商品属性 | 类别、价格、生命周期 | 商品系统 |
| 竞品动态 | 价格变化、促销 | 爬虫/API |

### 2.2 安全库存计算

**安全库存公式**：
```
安全库存 = Z值 × 标准差 × √平均提前期

Z值：服务水平对应值（95% = 1.65，99% = 2.33）
标准差：需求波动程度
平均提前期：从下单到到货的平均天数
```

**服务水平决策**：

| 服务水平 | Z值 | 断货风险 | 适用场景 |
|----------|-----|----------|----------|
| 90% | 1.28 | 10% | 低毛利商品 |
| 95% | 1.65 | 5% | 一般商品 |
| 99% | 2.33 | 1% | 核心商品 |

### 2.3 补货点计算

**再订货点（ROP）公式**：
```
ROP = (平均日销量 × 提前期) + 安全库存

示例：
平均日销量 = 10件
提前期 = 7天
安全库存 = 30件
ROP = (10 × 7) + 30 = 100件

当库存降至100件时触发补货
```

## 三、AI预测模型

### 3.1 时间序列模型

**常用算法**：

| 算法 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| ARIMA | 经典、稳定 | 季节性处理弱 | 短期预测 |
| Prophet | 处理季节性强 | 需要较多数据 | 季节性商品 |
| LSTM | 捕捉长期依赖 | 训练复杂 | 复杂模式 |
| XGBoost | 特征灵活 | 需要特征工程 | 多因素预测 |
| Transformer | 并行计算强 | 数据需求大 | 大规模预测 |

**Prophet示例**：
```python
from fbprophet import Prophet

# 数据准备
df = pd.read_csv('sales.csv')
df.columns = ['ds', 'y']

# 训练模型
model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    seasonality_mode='multiplicative'
)
model.add_regressor('promotion')  # 添加促销因子
model.fit(df)

# 预测未来30天
future = model.make_future_dataframe(periods=30)
future['promotion'] = [0, 1, 0, ...]  # 促销标记
forecast = model.predict(future)
```

Q4季节系数 = 1.5（旺季）
