# 第166号：库存优化管理指南

## 一、库存管理概述

### 1.1 库存管理重要性

| 维度 | 影响 | 优化目标 |
|-----|-----|---------|
| 资金占用 | 库存积压占用现金流 | 降低库存周转天数 |
| 仓储成本 | 仓储费用持续产生 | 优化库存结构 |
| 销售机会 | 缺货导致销售损失 | 保持合理库存 |
| 产品生命周期 | 过季/过期商品贬值 | 及时清理滞销品 |
| 供应链效率 | 影响发货速度 | 优化补货周期 |

### 1.2 库存管理KPI

```yaml
核心指标:

周转效率:
  库存周转率 = 销售成本 / 平均库存
  目标: >4次/年
  
  库存周转天数 = 365 / 库存周转率
  目标: <90天

服务水平:
  现货率 = 有货订单 / 总订单
  目标: >95%
  
  缺货率 = 缺货订单 / 总订单
  目标: <5%

库存健康:
  滞销库存占比
  目标: <15%
  
  库存准确率
  目标: >98%

效率指标:
  库存持有成本率
  目标: <25%/年
  
  仓储利用率
  目标: 70-85%
```

## 二、库存预测模型

### 2.1 需求预测方法
```python
# 需求预测模型示例

class DemandForecaster:
    """需求预测器"""
    
    def moving_average(self, sales, window=7):
        """移动平均法"""
        return np.convolve(sales, np.ones(window)/window, mode='valid')
    
    def exponential_smoothing(self, sales, alpha=0.3):
        """指数平滑法"""
        forecast = [sales[0]]
        for i in range(1, len(sales)):
            forecast.append(
                alpha * sales[i] + (1 - alpha) * forecast[-1]
            )
        return forecast
    
    def calculate_safety_stock(self, avg_demand, std, lead_time, service_level=0.95):
        """安全库存计算"""
        z_score = 1.96  # 95%服务水平的Z值
        return z_score * std * np.sqrt(lead_time)
    
    def calculate_reorder_point(self, avg_demand, lead_time, safety_stock):
        """再订货点"""
        return avg_demand * lead_time + safety_stock
```
### 2.2 库存补货策略

```yaml
补货策略:

定量订货法 (Q模型):
  触发条件: 库存降到再订货点
  订货量: 经济订货量(EOQ)
  
  EOQ = √(2 × 年需求 × 订货成本 / 单位存储成本)
  
  适用: 高价值、稳定需求商品

定期订货法 (P模型):
  触发条件: 固定周期检查
  订货量: 补至目标库存
  
  目标库存 = 平均日销 × (检查周期 + 提前期) + 安全库存
  
  适用: 低价值、可合并采购商品
```
### 2.3 ABC分类管理
```yaml
ABC分类:

A类商品 (高价值):
  特点:
    - 占库存价值70-80%
    - 占SKU数量10-20%
    - 严格管控
    - 小批量多频补货
  策略: 精准预测、安全库存充足

B类商品 (中等价值):
  特点:
    - 占库存价值15-25%
    - 占SKU数量20-30%
    - 常规管控
  策略: 标准预测、定期检查

C类商品(低价值):
  特点:
    - 占库存价值5-10%
    - 占SKU数量50-70%
    - 简化管控
  策略: 批量采购、定期清理
```
## 三、库存优化策略
### 3.1 滞销品处理
```yaml
处理流程:

识别滞销品:
  条件:
    - 库存 > 90天无销售
    - 周转率 < 2次/年
  - 库存 > 3个月销量
  处理方式:
    - 降价促销
    - 渠道转移
    - 退货供应商
    - 报废处理
处理时间线:
  第1-30天: 9折促销
  第31-60天: 7-8折促销
  第61-90天: 5折清仓
  第90天+: 报废处理
```
### 3.2 库存预警系统
```yaml
预警级别:

绿色预警 (正常):
  条件: 库存充足
  行动: 常规监控

黄色预警 (关注):
  条件: 库存 < 14天销量
  行动: 加快补货

橙色预警 (紧急):
  条件: 库存 < 7天销量
  行动: 立即补货

红色预警 (缺货):
  条件: 库存 < 3天销量
  行动: 紧急调货
```
## 四、仓库管理

仓库类型:
