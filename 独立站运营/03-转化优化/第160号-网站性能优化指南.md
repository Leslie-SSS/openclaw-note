# 第160号：网站性能优化指南

## 一、性能优化概述

### 1.1 性能优化价值

| 指标 | 性能影响 | 商业价值 |
|-----|---------|---------|
| 页面加载时间 | 每增加1秒 | 转化率下降7% |
| 首屏渲染 | 每延迟100ms | 跳出率增加1% |
| 交互响应 | 每增加0.1秒 | 用户满意度下降 |
| 移动端加载 | 超过3秒 | 53%用户离开 |

### 1.2 核心性能指标

```
Core Web Vitals (核心网页指标):

LCP (Largest Contentful Paint) - 最大内容绘制
├── 定义: 视口内最大元素渲染完成时间
├── 目标: < 2.5秒
├── 影响因素: 服务器响应、资源加载、渲染阻塞
└── 优化重点: 关键渲染路径

FID (First Input Delay) - 首次输入延迟
├── 定义: 用户首次交互到浏览器响应的时间
├── 目标: < 100毫秒
├── 影响因素: JavaScript执行、主线程阻塞
└── 优化重点: 代码分割、延迟执行

CLS (Cumulative Layout Shift) - 累积布局偏移
├── 定义: 页面视觉稳定性分数
├── 目标: < 0.1
├── 影响因素: 图片尺寸、动态内容、字体加载
└── 优化重点: 预留空间、稳定布局

INP (Interaction to Next Paint) - 交互到下一次绘制
├── 定义: 用户交互到页面更新的时间
├── 目标: < 200毫秒
├── 影响因素: 事件处理、渲染更新
└── 优化重点: 交互响应优化
```

### 3.4 字体优化

```yaml
字体优化策略:

格式选择:
  WOFF2: 最佳压缩率，现代浏览器支持
  WOFF: 广泛支持，中等压缩
  TTF/OTF: 兼容性好，体积较大
  EOT: 仅IE支持，已弃用

加载优化:
  - 子集化: 仅加载使用的字符
  - 预加载: <link rel="preload">
  - font-display: 控制加载行为
  - 本地存储: 缓存到localStorage

font-display选项:
  swap: 立即显示后备字体，加载后切换
  optional: 网络慢时使用后备字体
  block: 等待字体加载 (不推荐)
  fallback: 短暂等待，超时用后备

子集化示例:
  /* 只加载需要的字符 */
  @font-face {
    font-family: 'CustomFont';
    src: url('font-subset.woff2') format('woff2');
    unicode-range: U+0000-007F; /* 基本拉丁字符 */
  }
```

### 5.3 动画性能优化

```css
/* 动画性能优化 */

/* 1. 使用transform和opacity */
/* 这两个属性不触发布局重计算 */
.animate {
  transform: translateX(100px) scale(1.1);
  opacity: 0.8;
}

/* 2. 避免触发重绘的属性 */
/* 避免: width, height, top, left, margin, padding */

/* 3. 使用will-change提前告知浏览器 */
.will-animate {
  will-change: transform, opacity;
}

/* 动画结束后移除 */
.animation-complete {
  will-change: auto;
}

/* 4. 使用CSS动画代替JS动画 */
@keyframes slide-in {
  from {
    transform: translateX(-100%);
    opacity: 0;
  }
  to {
    transform: translateX(0);
    opacity: 1;
  }
}

.slide-in {
  animation: slide-in 0.3s ease-out;
}

/* 5. 使用requestAnimationFrame处理JS动画 */
function animate(element) {
  let start = null;
  const duration = 300;
  
  function step(timestamp) {
    if (!start) start = timestamp;
    const progress = (timestamp - start) / duration;
    
    if (progress < 1) {
      element.style.transform = `translateX(${progress * 100}px)`;
      requestAnimationFrame(step);
    }
  }
  
  requestAnimationFrame(step);
}

/* 6. 使用CSS contain隔离影响 */
.isolated-component {
  contain: layout paint;
}
```

```
┌─────────────────────────────────────────┐
│ 浏览器缓存 (Browser Cache)              │
│ TTL: 数秒到数年                         │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│ CDN缓存 (CDN Cache)                     │
│ TTL: 数小时到数年                       │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│ 反向代理缓存 (Nginx/Varnish)            │
│ TTL: 数秒到数小时                       │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│ 应用缓存 (Redis/Memcached)              │
│ TTL: 数秒到数小时                       │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│ 数据库 (MySQL/PostgreSQL)               │
│ 持久化存储                              │
└─────────────────────────────────────────┘

缓存策略:

Cache-Aside模式:
1. 先查缓存
2. 缓存命中: 返回数据
3. 缓存未命中: 查数据库，写入缓存

Write-Through模式:
1. 写入数据库同时写入缓存
2. 保持数据一致性

Write-Behind模式:
1. 先写缓存
2. 异步写入数据库
3. 高性能但可能丢数据

缓存更新策略:
- TTL过期: 简单，可能有脏数据
- 主动更新: 实时，但复杂
- 延迟双删: 解决并发问题
```

```javascript
// 1. Service Worker缓存
const CACHE_NAME = 'v1';
const urlsToCache = [
  '/',
  '/styles/main.css',
  '/scripts/main.js',
  '/images/logo.png',
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(urlsToCache))
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then((response) => {
        // 缓存命中，返回缓存
        if (response) {
          return response;
        }
        // 否则请求网络
        return fetch(event.request)
          .then((response) => {
            // 缓存新资源
            if (response.status === 200) {
              const responseClone = response.clone();
              caches.open(CACHE_NAME)
                .then((cache) => cache.put(event.request, responseClone));
            }
            return response;
          });
      })
  );
});
```

```javascript
// 1. 采集Core Web Vitals
function collectWebVitals() {
  // LCP
  new PerformanceObserver((entryList) => {
    const entries = entryList.getEntries();
    const lastEntry = entries[entries.length - 1];
    console.log('LCP:', lastEntry.startTime);
    sendToAnalytics('LCP', lastEntry.startTime);
  }).observe({ entryTypes: ['largest-contentful-paint'] });

  // FID
  new PerformanceObserver((entryList) => {
    const entries = entryList.getEntries();
    entries.forEach((entry) => {
      console.log('FID:', entry.processingStart - entry.startTime);
      sendToAnalytics('FID', entry.processingStart - entry.startTime);
    });
  }).observe({ entryTypes: ['first-input'] });

  // CLS
  let clsValue = 0;
  new PerformanceObserver((entryList) => {
    for (const entry of entryList.getEntries()) {
      if (!entry.hadRecentInput) {
        clsValue += entry.value;
      }
    }
    console.log('CLS:', clsValue);
  }).observe({ entryTypes: ['layout-shift'] });
}
```
