# 预测分析应用完整指南

## 一、理论框架

### 1.1 预测分析的定义与价值

预测分析（Predictive Analytics）是利用历史数据、统计算法和机器学习技术来预测未来结果的方法论。对于独立站而言，预测分析帮助从"发生了什么"进化到"将要发生什么"。

**核心价值：**

1. **销售预测**
   - 预测未来销售趋势
   - 指导库存规划
   - 优化营销节奏

2. **用户行为预测**
   - 预测用户流失
   - 预测购买意向
   - 预测用户价值

3. **需求预测**
   - 预测商品需求
   - 优化库存配置
   - 减少缺货/积压

4. **风险预测**
   - 预测欺诈风险
   - 预测退货风险
   - 预测支付风险

### 1.2 预测分析的方法分类

**按时间维度：**

| 类型 | 时间范围 | 应用场景 |
|-----|---------|---------|
| **短期预测** | 1-7天 | 日销售预测、库存补货 |
| **中期预测** | 1-3月 | 季度规划、促销规划 |
| **长期预测** | 3-12月 | 年度规划、战略决策 |

**按方法类型：**

| 方法 | 特点 | 适用场景 |
|-----|------|---------|
| **时间序列** | 基于历史趋势 | 销售、流量预测 |
| **回归分析** | 因果关系 | 价格、促销影响 |
| **分类模型** | 二分类/多分类 | 流失、购买预测 |
| **聚类分析** | 用户分群 | 用户价值预测 |
| **深度学习** | 复杂模式 | 推荐系统 |

### 1.3 预测分析的核心模型

**1. 时间序列模型**

适用于有规律趋势的数据预测：

```
销售数据示例：
Month    | Sales
---------|-------
Jan      | $100k
Feb      | $95k
Mar      | $110k
Apr      | $115k
May      | ?（预测）

常用模型：
- 移动平均（Moving Average）
- 指数平滑（Exponential Smoothing）
- ARIMA
- Prophet（Facebook开源）
```

**2. 用户流失预测模型**

基于用户行为预测流失概率：

```
输入特征：
- 最近一次购买时间
- 购买频率
- 消费金额
- 网站访问频率
- 邮件打开率
- 客服投诉次数

输出：
- 流失概率（0-100%）
- 高风险用户列表

常用模型：
- 逻辑回归
- 随机森林
- XGBoost
```

**3. 用户终身价值预测（CLV）**

预测用户未来价值：

```
简单模型：
CLV = 平均订单价值 × 购买频率 × 客户生命周期

高级模型：
- BG/NBD模型（概率模型）
- 机器学习模型
- 深度学习模型
```

### 1.4 预测分析的评估指标

**回归模型评估：**

| 指标 | 含义 | 计算方式 |
|-----|------|---------|
| **MAE** | 平均绝对误差 | Σ|实际值-预测值|/n |
| **RMSE** | 均方根误差 | √(Σ(实际值-预测值)²/n) |
| **MAPE** | 平均绝对百分比误差 | Σ|误差/实际值|/n |

**分类模型评估：**

| 指标 | 含义 |
|-----|------|
| **准确率** | 正确预测比例 |
| **精确率** | 预测为正的准确度 |
| **召回率** | 实际为正的识别率 |
| **AUC-ROC** | 模型区分能力 |

**步骤三：模型训练（Prophet示例）**

```python
from prophet import Prophet

# 准备数据格式
df_prophet = df.rename(columns={'date': 'ds', 'sales': 'y'})

# 创建并训练模型
model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False
)
model.add_country_holidays(country_name='US')
model.fit(df_prophet)

# 预测未来30天
future = model.make_future_dataframe(periods=30)
forecast = model.predict(future)

# 查看预测结果
print(forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].tail(30))
```

### 2.4 用户终身价值预测实操

**简单方法：RFM分群**

```python
# 计算RFM
import datetime

today = datetime.datetime.now()

rfm = df.groupby('user_id').agg({
    'order_date': lambda x: (today - x.max()).days,  # Recency
    'order_id': 'count',  # Frequency
    'order_amount': 'sum'  # Monetary
}).rename(columns={
    'order_date': 'recency',
    'order_id': 'frequency',
    'order_amount': 'monetary'
})

# 分段打分
rfm['r_score'] = pd.qcut(rfm['recency'], 5, labels=[5,4,3,2,1])
rfm['f_score'] = pd.qcut(rfm['frequency'], 5, labels=[1,2,3,4,5])
rfm['m_score'] = pd.qcut(rfm['monetary'], 5, labels=[1,2,3,4,5])

rfm['rfm_score'] = rfm['r_score'].astype(str) + rfm['f_score'].astype(str) + rfm['m_score'].astype(str)

# 预测CLV
# 高RFM分数 = 高CLV
```

**高级方法：BG/NBD + Gamma-Gamma模型**

```python
from lifetimes import BetaGeoFitter, GammaGammaFitter

# 准备数据
data = summary_data_from_transaction_data(
    df,
    'user_id',
    'order_date',
    observation_period_end=today,
    freq='D'
)

# BG/NBD模型（预测购买次数）
bgf = BetaGeoFitter(penalizer_coef=0.0)
bgf.fit(data['frequency'], data['recency'], data['T'])

# 预测未来30天购买次数
data['predicted_purchases'] = bgf.conditional_expected_number_of_purchases_up_to_time(
    30, data['frequency'], data['recency'], data['T']
)

# Gamma-Gamma模型（预测金额）
ggf = GammaGammaFitter(penalizer_coef=0.0)
ggf.fit(data['frequency'], data['monetary_value'])

# 预测CLV
data['clv'] = ggf.customer_lifetime_value(
    bgf,
    data['frequency'],
    data['recency'],
    data['T'],
    data['monetary_value'],
    time=12,  # 12个月
    discount_rate=0.01
)
```

### 4.1 案例一：库存需求预测

**背景：**
某服装独立站，库存周转率低，经常断货或积压。

**目标：**
预测未来14天各SKU需求，指导补货。

**实施过程：**

**1. 数据收集**
```
历史数据（2年）：
- 每日销售数据
- SKU信息
- 价格变化
- 促销活动
```

**2. 模型开发**
```
使用Prophet + 天气数据 + 促销数据
训练周期：3个月
测试周期：1个月
```

**3. 模型效果**
```
MAPE: 18%
畅销SKU: MAPE 12%
长尾SKU: MAPE 25%
```

**4. 应用效果**
```
实施6个月后：
- 断货率：15% → 5%
- 库存周转率：+35%
- 库存成本：-20%
```

### 4.2 案例二：用户流失预警

**背景：**
某订阅制美妆盒子，月流失率8%，希望提前识别流失用户。

**实施过程：**

**1. 定义流失**
```
流失：连续2个月未续订
```

**2. 特征工程**
