# 第158号：用户生命周期管理指南

## 一、用户生命周期概述

### 1.1 生命周期定义

用户生命周期（Customer Lifecycle）是指用户从首次接触品牌到最终流失的完整旅程，包含认知、获取、激活、留存、变现、推荐等阶段。

### 1.2 生命周期价值模型

```
用户生命周期价值 (CLV/LTV) 计算公式:

基础公式:
LTV = ARPU × 用户生命周期时长

进阶公式:
LTV = (平均客单价 × 购买频次 × 毛利率) ÷ 流失率

详细公式:
LTV = Σ(期间收入 - 期间成本) × 留存率^n
      ─────────────────────────────
              (1 + 折现率)^n

关键变量:
├── ARPU: 每用户平均收入
├── AOV: 平均订单价值
├── 购买频次: 年均购买次数
├── 毛利率: 产品毛利比例
├── 留存率: 用户持续活跃比例
└── 流失率: 用户离开比例
```

### 1.3 生命周期阶段划分

| 阶段 | 定义 | 关键指标 | 运营重点 | 时间周期 |
|-----|-----|---------|---------|---------|
| 认知期 | 首次接触品牌 | 曝光量、CTR | 品牌曝光 | 0-7天 |
| 获取期 | 完成注册/首访 | 注册率、获客成本 | 降低门槛 | 0-14天 |
| 激活期 | 完成核心行为 | 激活率、首次转化 | 价值体验 | 1-30天 |
| 留存期 | 持续使用/复购 | 留存率、活跃度 | 习惯培养 | 30-90天 |
| 变现期 | 产生商业价值 | ARPU、LTV | 价值最大化 | 90天+ |
| 推荐期 | 主动推荐品牌 | NPS、裂变率 | 口碑传播 | 持续 |
| 流失期 | 停止使用/购买 | 流失率 | 召回挽回 | 预警期 |

## 三、RFM用户分层模型

### 3.1 RFM模型详解

```
RFM模型定义:

R (Recency) - 最近一次购买时间
F (Frequency) - 购买频率
M (Monetary) - 消费金额

评分标准:

R评分:
├── 5分: 7天内购买
├── 4分: 8-14天购买
├── 3分: 15-30天购买
├── 2分: 31-60天购买
└── 1分: 60天以上

F评分:
├── 5分: 年购买10次以上
├── 4分: 年购买6-9次
├── 3分: 年购买3-5次
├── 2分: 年购买2次
└── 1分: 年购买1次

M评分:
├── 5分: 年消费5000元以上
├── 4分: 年消费2000-4999元
├── 3分: 年消费1000-1999元
├── 2分: 年消费500-999元
└── 1分: 年消费500元以下
```

### 3.3 RFM分析SQL

```sql
-- RFM分析SQL

WITH rfm_calc AS (
    SELECT 
        user_id,
        DATEDIFF(CURRENT_DATE, MAX(order_date)) as recency,
        COUNT(DISTINCT order_id) as frequency,
        SUM(amount) as monetary
    FROM orders
    WHERE order_date >= DATE_SUB(CURRENT_DATE, INTERVAL 1 YEAR)
    GROUP BY user_id
),
rfm_score AS (
    SELECT 
        user_id,
        recency,
        frequency,
        monetary,
        -- R评分
        CASE 
            WHEN recency <= 7 THEN 5
            WHEN recency <= 14 THEN 4
            WHEN recency <= 30 THEN 3
            WHEN recency <= 60 THEN 2
            ELSE 1
        END as r_score,
        -- F评分
        CASE 
            WHEN frequency >= 10 THEN 5
            WHEN frequency >= 6 THEN 4
            WHEN frequency >= 3 THEN 3
            WHEN frequency >= 2 THEN 2
            ELSE 1
        END as f_score,
        -- M评分
        CASE 
            WHEN monetary >= 5000 THEN 5
            WHEN monetary >= 2000 THEN 4
            WHEN monetary >= 1000 THEN 3
            WHEN monetary >= 500 THEN 2
            ELSE 1
        END as m_score
    FROM rfm_calc
)
SELECT 
    user_id,
    r_score,
    f_score,
    m_score,
    CONCAT(r_score, f_score, m_score) as rfm_group,
    CASE 
        WHEN r_score >= 4 AND f_score >= 4 AND m_score >= 4 
            THEN '重要价值用户'
        WHEN r_score >= 4 AND f_score >= 3 
            THEN '重要发展用户'
        WHEN r_score >= 4 AND m_score >= 3 
            THEN '重要保持用户'
        WHEN r_score <= 2 AND f_score >= 3 AND m_score >= 3 
            THEN '重要挽留用户'
        WHEN r_score >= 3 AND f_score >= 2 AND m_score >= 2 
            THEN '一般价值用户'
        WHEN r_score >= 3 AND f_score <= 2 
            THEN '一般发展用户'
        ELSE '低价值用户'
    END as user_segment
FROM rfm_score;
```

```yaml
      - 发送补货提醒

      - 发送升级通知

  预警触发:
    条件: 30天未登录 OR 60天未购买

      - 打标签: 流失风险

      - 打标签: 高流失风险
      - 客服电话回访
      - 专属回归礼包
  
  流失确认:
    条件: 90天未活跃
    动作:
      - 标记为流失用户
      - 加入定期召回列表
      - 分析流失原因
```

## 七、行业案例分析

### 7.1 电商平台案例

```
案例: 某服装电商

背景:
- 年GMV: 5000万
- 用户数: 50万
- 复购率: 18%
- LTV: ¥280

问题诊断:
- 新用户激活率低 (25%)
- 首购后流失严重
- 复购周期过长

解决方案:

1. 新用户优化
   - 注册即送50元优惠券包
   - 新手7天任务系统
   - 首购推荐热门套装
   - 3天内未购追加优惠

2. 激活期优化
   - 购买后1天: 搭配推荐
   - 购买后3天: 洗护指南
   - 购买后7天: 晒单激励
   - 购买后14天: 复购优惠

3. 复购促进
   - 基于购买周期的复购提醒
   - 会员等级体系
   - 积分商城
   - 季度会员日

实施效果:
- 激活率: 25% → 45%
- 次月复购率: 8% → 18%
- LTV: ¥280 → ¥420
- 用户生命周期: 6月 → 10月
```
