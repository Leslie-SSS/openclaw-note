### 2. 关键支付指标

**成功率指标**：
```
授权率（Authorization Rate）：
- 定义：成功授权/总请求
- 行业标准：85-95%
- 优秀：>95%

捕获率（Capture Rate）：
- 定义：成功捕获/成功授权
- 行业标准：95-99%

整体成功率：
- 定义：最终成功/总请求
- 计算公式：授权率 × 捕获率
- 目标：>90%
```

**成本指标**：
```
支付成本率：
- 定义：支付费用/交易金额
- 行业平均：2-3%
- 优秀：<2%

拒付率：
- 定义：拒付金额/总交易
- 行业标准：<1%
- 卡组织要求：<0.9%

退款率：
- 定义：退款金额/总交易
- 行业标准：<5%
```

**性能指标**：
```
支付处理时间：
- 定义：从请求到确认
- 目标：<3秒

结算周期：
- 定义：从交易到到账
- 目标：T+0到T+3
```

### 5. 支付方式分析

**支付方式绩效**：
```sql
-- 支付方式分析查询
SELECT 
    payment_method,
    COUNT(*) as transaction_count,
    SUM(amount) as total_amount,
    AVG(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success_rate,
    AVG(processing_time) as avg_time,
    SUM(fee) / SUM(amount) as fee_rate,
    AVG(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 
    (1 - SUM(fee)/SUM(amount)) as efficiency_score
FROM payment_facts
WHERE created_at >= DATE_SUB(CURRENT_DATE, INTERVAL 30 DAY)
GROUP BY payment_method
ORDER BY total_amount DESC;
```

def calculate_fraud_score(transaction):
    score = 0
    
    # 规则评分
    if transaction['amount'] > 1000:
        score += 20
    
    if transaction['country'] != transaction['ip_country']:
        score += 30
    
    if transaction['velocity_24h'] > 5:
        score += 25
    
    if transaction['is_new_device']:
        score += 15

        score += 40
