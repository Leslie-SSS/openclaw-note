# 385. 支付安全防护

## 核心要点

### 1. 支付安全威胁类型

**欺诈（Fraud）**：
- 被盗卡欺诈（Card-not-present fraud）
- 身份盗用
- 账户接管
- 友善欺诈（虚假拒付）

**数据泄露**：
- 支付信息被盗
- 用户数据泄露
- 数据库入侵

**网络攻击**：
- DDoS攻击
- 中间人攻击
- 钓鱼攻击
- 恶意软件

**内部威胁**：
- 员工滥用权限
- 数据泄露
- 系统破坏

### 2. 支付安全标准

**PCI DSS**（支付卡行业数据安全标准）：
```
12项核心要求：
1. 安装和维护防火墙
2. 不使用默认密码
3. 保护存储的持卡人数据
4. 加密传输中的持卡人数据
5. 使用并更新防病毒软件
6. 开发和维护安全系统
7. 按需限制访问
8. 为每个用户分配唯一ID
9. 限制对持卡人数据的物理访问
10. 跟踪和监控所有访问
11. 定期测试安全系统和流程
12. 维护信息安全政策
```

**合规等级**：
```
Level 1：年交易量 > 600万笔
- 年度现场审计
- 季度网络扫描

Level 2：100-600万笔
- 年度自我评估
- 季度网络扫描

Level 3：2-100万笔
- 年度自我评估
- 季度网络扫描

Level 4：< 2万笔
- 年度自我评估
```

**其他标准**：
- ISO 27001：信息安全管理体系
- SOC 2：服务组织控制报告
- GDPR：数据保护

### 3. 欺诈类型和识别

**常见欺诈类型**：
```
1. 被盗卡欺诈
   特征：新账户、大额、加急配送
   比例：占电商欺诈70%

2. 身份盗用
   特征：使用他人信息开户
   比例：15%

3. 账户接管
   特征：登录异常、修改信息
   比例：10%

4. 友善欺诈
   特征：收到商品后拒付
   比例：5%
```

**欺诈指标**：
```
高风险信号：
✓ 首次购买客户
✓ 大额订单（>$500）
✓ 加急配送
✓ 账单地址≠配送地址
✓ 多次尝试不同卡
✓ IP地址与账单地址不符
✓ 免费邮箱
✓ 可疑设备
```

```javascript
// 欺诈规则配置
const fraudRules = {

    highRiskCountry: {

    },
    
    // 规则4：速度异常
    velocityCheck: {
        condition: 'orders_last_24h > 3',
        score: 50,
        action: 'block'
    },
    
    // 规则5：设备指纹
    deviceRisk: {
        condition: 'device_risk_score > 70',
        score: 35,
        action: 'challenge'
    }
};

// 评分逻辑
function calculateFraudScore(transaction) {
    let totalScore = 0;
    const triggeredRules = [];
    
    for (const [ruleName, rule] of Object.entries(fraudRules)) {
        if (evaluateCondition(rule.condition, transaction)) {
            totalScore += rule.score;
            triggeredRules.push(ruleName);
        }
    }
    
    // 决策
    let decision = 'approve';
    if (totalScore > 70) decision = 'block';
    else if (totalScore > 40) decision = 'review';
    else if (totalScore > 20) decision = 'challenge';
    
    return {
        score: totalScore,
        decision: decision,
        rules: triggeredRules
    };
}
```

### 2. 3D Secure（3DS）

**3DS 2.0优势**：
```
对比3DS 1.0：
- 用户体验更好（无跳转）
- 数据传输更多（风险评估更准）
- 支持移动端
- 豁免机制（低风险免验证）

核心概念：
- 强客户认证（SCA）
- 风险豁免
- 挑战流程
- 认证结果
```

**实施策略**：
```javascript
// 3DS实施
async function process3DS(payment) {
    // 1. 评估是否需要3DS
    const exemption = await checkExemption(payment);
    
    if (exemption.allowed) {
        // 豁免3DS
        return processPayment(payment);
    }
    
    // 2. 发起3DS认证
    const auth = await initiate3DS(payment);
    
    if (auth.status === 'challenge_required') {
        // 需要用户验证
        const challengeResult = await showChallenge(auth);
        return challengeResult;
    } else {
        // 无摩擦认证
        return processPayment(payment, auth);
    }
}

// 豁免规则
function checkExemption(payment) {
    // 低风险交易
    if (payment.fraud_score < 20) {
        return { allowed: true, reason: 'low_risk' };
    }
    
    // 小额交易
    if (payment.amount < 30) {
        return { allowed: true, reason: 'low_value' };
    }
    
    // 白名单客户
    if (payment.customer.whitelisted) {
        return { allowed: true, reason: 'trusted_customer' };
    }
    
    return { allowed: false };
}
```

### 4. 地址验证（AVS）

**AVS检查**：
```javascript
// 地址验证服务
async function verifyAddress(payment) {
    const result = {
        match: false,
        zipMatch: false,
        addressMatch: false,
        score: 0
    };
    
    // 调用银行AVS
    const avsResponse = await callAVS({
        address: payment.billingAddress,
        zip: payment.billingZip,
        cardNumber: payment.cardNumber
    });
    
    // 解析结果
    switch (avsResponse.code) {
        case 'Y':  // 完全匹配
            result.match = true;
            result.zipMatch = true;
            result.addressMatch = true;
            result.score = 100;
            break;
            
        case 'A':  // 地址匹配，邮编不匹配
            result.addressMatch = true;
            result.score = 50;
            break;
            
        case 'Z':  // 邮编匹配，地址不匹配
            result.zipMatch = true;
            result.score = 50;
            break;
            
        case 'N':  // 都不匹配
            result.score = 0;
            break;
    }
    
    return result;
}

// AVS决策
function avsDecision(avsResult) {
    if (avsResult.score === 100) {
        return 'approve';
    } else if (avsResult.score >= 50) {
        return 'review';
    } else {
        return 'reject';
    }
}
```

### 5. CVV验证

**CVV规则**：
```
CVV验证：
- 禁止存储CVV（PCI DSS要求）
- 仅用于交易时验证
- 不匹配则拒绝

验证流程：
1. 用户输入CVV
2. 发送到发卡行验证
3. 验证结果返回
4. 不匹配则拒绝交易

注意事项：
- CVV不存储
- 不在日志中记录
- 传输必须加密
```

```python
# 异常交易检测
def detect_anomalies(transactions):
    anomalies = []
    
    # 1. 统计异常
    avg_amount = transactions['amount'].mean()
    std_amount = transactions['amount'].std()
    
    for tx in transactions:
        z_score = (tx['amount'] - avg_amount) / std_amount
        if abs(z_score) > 3:  # 3个标准差
            anomalies.append({
                'transaction': tx,
                'reason': 'amount_anomaly',
                'z_score': z_score
            })
    
    # 2. 行为异常
    user_history = get_user_history(tx['user_id'])
    if tx['amount'] > user_history['avg'] * 5:
        anomalies.append({
            'transaction': tx,
            'reason': 'unusual_amount_for_user'
        })
    
    # 3. 地理异常
    if distance(tx['ip_location'], tx['billing_address']) > 1000:  # 1000km
        anomalies.append({
            'transaction': tx,
            'reason': 'geo_mismatch'
        })
    
    return anomalies
```
