## 三、用户画像构建流程

### 3.1 数据采集

**数据源清单**：

1. **第一方数据**（核心）
   - 网站行为数据（GA4/自定义追踪）
   - 交易数据（Shopify/WooCommerce）
   - 邮件互动数据（打开、点击）
   - 客服对话数据
   - 问卷调研数据

2. **第二方数据**
   - 社交媒体互动
   - 合作伙伴数据

3. **第三方数据**（补充）
   - 人口统计数据
   - 兴趣标签数据
   - 信用评分数据

**技术实现**：

```javascript
// 网站行为追踪示例
track('product_view', {
  product_id: 'SKU123',
  category: 'dresses',
  price: 89.99,
  color: 'black',
  size: 'M',
  timestamp: Date.now()
});

track('add_to_cart', {
  product_id: 'SKU123',
  quantity: 1,
  price: 89.99
});

track('purchase', {
  order_id: 'ORD456',
  products: [...],
  total: 95.98,
  discount: 10
});
```

### 3.2 数据整合与清洗

**数据清洗步骤**：

1. **去重**：合并同一用户多设备数据
2. **补全**：推断缺失的属性值
3. **标准化**：统一数据格式
4. **验证**：剔除异常值

**用户识别方法**：

| 场景 | 识别方式 | 准确度 |
|------|----------|--------|
| 已登录 | User ID | 100% |
| 未登录-有邮箱 | Email Hash | 90% |
| 未登录-无邮箱 | Device Fingerprint | 60-80% |

### 3.3 特征工程

**核心特征计算**：

**RFM模型**：
- Recency：最近购买距今天数
- Frequency：过去365天购买次数
- Monetary：过去365天消费总额

**RFM分群矩阵**：

| 分群 | R | F | M | 特征 | 策略 |
|------|---|---|---|------|------|
| 重要价值 | 高 | 高 | 高 | 最近购买+高频+高消费 | VIP维护 |
| 重要发展 | 高 | 低 | 高 | 最近购买+低频+高消费 | 提频策略 |
| 重要保持 | 低 | 高 | 高 | 较久未购+高频+高消费 | 召回策略 |
| 重要挽留 | 低 | 低 | 高 | 较久未购+低频+高消费 | 强力召回 |
| 一般价值 | 高 | 高 | 低 | 最近购买+高频+低消费 | 提升客单 |
| 一般发展 | 高 | 低 | 低 | 新客 | 培养习惯 |
| 一般保持 | 低 | 高 | 低 | 活跃度下降 | 激活策略 |
| 低价值 | 低 | 低 | 低 | 沉睡/流失 | 低成本维护 |

**生命周期阶段**：

1. 潜客（未购买）
2. 新客（首购<30天）
3. 活跃客（30天内有购买）
4. 沉睡客（30-90天未购）
5. 流失客（>90天未购）

### 3.4 标签体系

**标签分类**：

1. **事实标签**（客观属性）
   - 性别：男/女
   - 地区：美国加州
   - 会员等级：Gold

2. **统计标签**（聚合计算）
   - 客单价区间：$50-100
   - 购买频率：月均2次
   - 品类偏好：连衣裙

3. **预测标签**（模型推断）
   - 流失风险：高/中/低
   - LTV预测：$200-300
   - 下单概率：75%

## 四、用户画像应用场景

### 4.1 精准营销

### 4.4 客服个性化

**应用示例**：
- VIP客户优先服务
- 高退货率客户加强沟通
- 大额订单客户专属跟进

## 八、关键案例：Spotify用户画像实践

### 背景介绍

Spotify拥有5亿+用户，通过精细的用户画像实现高度个性化体验。

### 画像维度

**行为维度**：
- 听歌频率、时长、时段
- 播放列表创建
- 歌曲跳过率
- 搜索行为
- 分享行为

**偏好维度**：
- 流派偏好（基于播放数据）
- 艺人偏好（关注、播放）
- 情境偏好（运动、睡眠、工作）
- 语言偏好

**社交维度**：
- 关注的用户/艺人
- 分享频率
- 协作播放列表

### 应用场景

1. **个性化推荐**
   - Discover Weekly（每周发现）
   - Daily Mix（每日混合）
   - Release Radar（新歌雷达）

2. **精准营销**
   - 基于听歌习惯的演出推荐
   - 基于流派的会员转化
   - 基于活跃度的流失预防

3. **产品优化**
   - 基于用户行为的UI优化
   - 基于偏好的功能优先级

### 核心成果

| 指标 | 效果 |
|------|------|
| 推荐准确率 | >80% |
| 用户参与度 | +40% |
| 流失率降低 | 25% |
| 会员转化率 | +30% |

### 可复用经验

1. **行为数据优先**：行为比声明更真实
2. **实时更新**：用户画像动态演进
3. **多维度交叉**：单一维度不够精准
4. **隐私友好**：在合规前提下最大化价值
5. **闭环优化**：画像效果持续评估

## 总结

用户画像是数据驱动营销的基础设施。关键要点：

1. **数据质量第一**：准确、完整、及时
2. **业务导向**：从应用场景反推画像设计
3. **动态更新**：用户行为实时反映到画像
4. **隐私合规**：在合规框架内使用数据
5. **持续迭代**：画像体系随业务演进

投入产出比通常在3-6个月回本，是独立站数据化运营的核心能力。
