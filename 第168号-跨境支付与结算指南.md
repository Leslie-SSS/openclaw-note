## 二、支付服务商选择

### 2.1 支付服务商对比

| 服务商 | 优势 | 劣势 | 手续费 | 适用场景 |
|-------|-----|-----|--------|---------|
| Stripe | 功能全、API友好 | 部分地区不支持 | 2.9%+$0.3 | 全球电商 |
| PayPal | 品牌认知度高 | 手续费高 | 3.5%+固定 | 全球电商 |
| Adyen | 企业级、功能强 | 入门门槛高 | 定制 | 大型企业 |
| Checkout.com | 覆盖广 | 服务一般 | 2.9%+ | 跨境电商 |
| 2Checkout | 覆盖广 | 用户体验一般 | 3.5% | 中小企业 |
| Airwallex | 费率低 | 功能较新 | 1.8%+ | 中国出海 |

### 2.2 选择决策树

```
支付服务商选择:

├── 月交易额 < $50,000
│   ├── 主流市场 (美/欧)
│   │   └── 推荐: Stripe + PayPal
│   └── 新兴市场
│       └── 推荐: 本地支付 + PayPal
│
├── 月交易额 $50,000-500,000
│   ├── 功能需求高
│   │   └── 推荐: Stripe + Adyen
│   └── 成本敏感
│       └── 推荐: Airwallex + Stripe
│
└── 月交易额 > $500,000
    └── 推荐: 自建收单 + 多服务商
```

## 三、支付风险管理

### 3.1 欺诈防护

```yaml
欺诈类型:

卡片欺诈:
  - 盗刷
  - 伪造卡
  - 测试卡号

账户欺诈:
  - 账号盗用
  - 身份冒用

商家欺诈:
  - 虚假交易
  - 刷单

防护措施:

事前预防:
  - 地址验证 (AVS)
  - CVV验证
  - 3D Secure
  - IP检测
  - 设备指纹

事中监控:
  - 实时风控规则
  - 异常行为检测
  - 黑名单过滤
  - 金额限制

事后处理:
  - 人工审核
  - 拒付应对
  - 证据收集
```

### 3.2 拒付管理

```yaml
拒付类型:

欺诈拒付:
  - 非授权交易
  - 盗卡使用

商业拒付:
  - 未收到货
  - 商品与描述不符
  - 退货未退款

技术拒付:
  - 重复扣款
  - 金额错误

拒付应对流程:

收到拒付通知:
  1. 立即响应 (通常7天内)
  2. 收集证据
     - 订单信息
     - 物流证明
     - 签收记录
     - 通信记录
  3. 准备抗辩材料
  4. 提交抗辩
  5. 跟进结果

预防措施:
  - 清晰的产品描述
  - 及时的物流更新
  - 完善的客服
  - 证据保存
```

## 四、结算与对账

```yaml

  - 常规: 2天
  - 可申请: 次日

  - 常规: 即时可用

  - 通常: 3-5天
```

## 五、汇率管理

### 5.1 汇率策略

```yaml
定价策略:

本地货币定价:
  优点:
    - 用户体验好
    - 转化率高
  缺点:
    - 汇率风险
    - 定价复杂

美元定价:
  优点:
    - 管理简单
    - 无汇率风险
  缺点:
    - 用户体验差
    - 转化率低

混合策略:
  - 主要市场: 本地货币
  - 其他市场: 美元

汇率对冲:
  - 远期结售汇
  - 多币种账户
  - 定期调价
```
