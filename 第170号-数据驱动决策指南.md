## 二、指标体系设计

### 2.1 北极星指标

```yaml
北极星指标选择:

定义: 最能反映业务核心价值的单一指标

选择原则:
  - 反映用户价值
  - 可衡量
  - 可行动
  - 与业务目标一致

示例:
  电商平台: GMV / 订单量
  订阅产品: 活跃订阅用户数
  内容平台: 日活跃用户数
  SaaS产品: ARR (年度经常性收入)

避免:
  - 虚荣指标 (PV、粉丝数)
  - 短期指标 (日收入)
  - 不可控行动指标
```

### 2.2 指标层级

```yaml
指标金字塔:

第一层 - 北极星指标:
  - 核心业务指标
  - 1个
  - 示例: GMV

第二层 - 关键指标:
  - 直接影响北极星
  - 3-5个
  - 示例: 订单量、客单价、转化率

第三层 - 支撑指标:
  - 影响关键指标
  - 10-20个
  - 示例: 流量、加购率、支付成功率

第四层 - 过程指标:
  - 日常监控
  - 数十个
  - 示例: 页面加载时间、客服响应时间
```

### 2.3 OMTM模型

```yaml
OMTM (One Metric That Matters):

定义: 在特定时期内最需要关注的单一指标

选择依据:
  - 业务阶段
  - 当前挑战
  - 战略重点

示例:
  启动期:
    指标: 新用户获取
    关注: 注册量、首购率
  
  成长期:
    指标: 用户增长
    关注: DAU、MAU、增长率
  
  成熟期:
    指标: 用户留存
    关注: 留存率、复购率、LTV
  
  变现阶段:
    指标: 盈利能力
    关注: 毛利率、净利率、ROI
```

## 三、数据分析方法

### 3.1 描述性分析

```yaml
描述性分析:

目的: 了解发生了什么

方法:
  趋势分析:
    - 日/周/月趋势
    - 同比/环比
    - 增长率
  
  分布分析:
    - 用户分布
    - 订单分布
    - 地域分布
  
  构成分析:
    - 渠道构成
    - 品类构成
    - 用户构成

示例报表:
  - 日报: 核心指标日变化
  - 周报: 周度趋势分析
  - 月报: 月度全面复盘
```

### 3.2 诊断性分析

```yaml
诊断性分析:

目的: 了解为什么发生

方法:
  下钻分析:
    - 维度下钻
    - 层层深入
    - 找到根本原因
  
  相关性分析:
    - 指标关联
    - 相关系数
    - 因果验证
  
  异常分析:
    - 异常检测
    - 原因追溯
    - 影响评估

示例场景:
  销售下降分析:
    1. 哪个渠道下降?
    2. 哪个品类下降?
    3. 哪个用户群下降?
    4. 具体原因是什么?
```

### 3.3 预测性分析

```yaml
预测性分析:

目的: 预测未来会发生什么

方法:
  时间序列:
    - 移动平均
    - 指数平滑
    - ARIMA
  
  回归分析:
    - 线性回归
    - 多元回归
    - 逻辑回归
  
  机器学习:
    - 分类模型
    - 预测模型
    - 深度学习

应用场景:
  - 销售预测
  - 库存预测
  - 用户流失预测
  - LTV预测
```

## 四、数据看板设计

### 4.1 看板层级

```yaml
看板分层:

高管看板:
  内容: 核心业务指标
  频率: 日更新
  形式: 简洁、直观
  示例:
    - GMV
    - 订单量
    - 用户数
    - 毛利率

运营看板:
  内容: 运营指标
  频率: 实时/小时
  形式: 详细、多维
  示例:
    - 实时流量
    - 转化漏斗
    - 渠道表现
    - 活动效果

专项看板:
  内容: 特定领域
  频率: 按需
  形式: 专业、深度
  示例:
    - 广告投放
    - 用户行为
    - 商品分析
    - 库存分析
```

### 4.2 看板设计原则

```yaml
设计原则:

简洁:
  - 核心指标突出
  - 避免信息过载
  - 一页一主题

直观:
  - 可视化优先
  - 颜色区分
  - 趋势清晰

行动:
  - 指标可行动
  - 异常预警
  - 下钻能力

实时:
  - 数据及时
  - 自动更新
  - 推送提醒
```
