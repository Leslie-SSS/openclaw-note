## 1. 关键指标定义和计算方法

### 1.1 获客成本（CAC - Customer Acquisition Cost）

**定义**：获取一个付费客户所需的平均成本

**计算公式**：
```
CAC = 总营销费用 + 总销售费用 / 新增付费客户数
```

**示例**：
- 1月份营销支出：$50,000
- 销售团队成本：$20,000
- 新增付费客户：1,000人
- **CAC = ($50,000 + $20,000) / 1,000 = $70**

**细分指标**：
- **渠道CAC**：按广告渠道计算（Facebook CAC、Google CAC、TikTok CAC等）
- **产品线CAC**：不同产品线的获客成本
- **时段CAC**：按月/季度/活动周期计算

**优化建议**：
- CAC应小于LTV的1/3（理想比例 LTV:CAC = 3:1）
- 追踪CAC回收周期（Payback Period）
- 结合ROAS评估广告效率

---

### 1.2 客户终身价值（LTV - Lifetime Value / CLV）

**定义**：客户在整个生命周期内为企业创造的总利润

**基础计算公式**：
```
LTV = 平均客单价 × 购买频率 × 客户生命周期（月/年）
```

**进阶计算公式**（考虑利润率）：
```
LTV = (平均客单价 × 购买频率 × 毛利率) × 客户生命周期
```

**SaaS/订阅模式公式**：
```
LTV = ARPU（每用户平均收入）/ Churn Rate（流失率）
```

**详细计算示例**：
```
某服装独立站数据：
- 平均客单价（AOV）：$85
- 年均购买次数：3.5次
- 平均客户生命周期：2.5年
- 毛利率：45%

LTV = $85 × 3.5 × 0.45 × 2.5 = $334.69
```

**LTV提升策略**：
1. **提高客单价**：捆绑销售、满减活动
2. **增加购买频率**：会员计划、邮件营销、再营销
3. **延长生命周期**：优质客户服务、忠诚度计划
4. **提升留存率**：个性化推荐、客户关怀

---

### 1.3 广告支出回报率（ROAS - Return on Ad Spend）

**定义**：每花费1元广告费产生的收入

**计算公式**：
```
ROAS = 广告带来的收入 / 广告费用
```

**表示方式**：
- 比例形式：ROAS 4:1（每花$1获得$4收入）
- 百分比形式：400%
- 小数形式：4.0

**示例**：
```
Facebook广告投放：
- 广告支出：$10,000
- 产生的收入：$45,000
- ROAS = $45,000 / $10,000 = 4.5 或 450%
```

**盈亏平衡ROAS**：
```
盈亏平衡ROAS = 1 / 毛利率

例：毛利率40%
盈亏平衡ROAS = 1 / 0.4 = 2.5

意味着ROAS > 2.5才能盈利
```

**ROAS vs ROI**：
| 指标 | 计算基础 | 适用场景 |
|------|----------|----------|
| ROAS | 收入 | 广告投放优化 |
| ROI | 利润 | 整体投资决策 |

**ROAS基准**（电商行业参考）：
- **优秀**：> 400%
- **良好**：300% - 400%
- **及格**：200% - 300%
- **需优化**：< 200%

---

### 1.4 平均客单价（AOV - Average Order Value）

**定义**：每笔订单的平均金额

**计算公式**：
```
AOV = 总收入 / 订单数量
```

**示例**：
```
月度数据：
- 总收入：$125,000
- 订单数：1,500
- AOV = $125,000 / 1,500 = $83.33
```

**提升AOV的策略**：
1. **产品捆绑**：套餐组合、买二送一
2. **满额优惠**：满$100减$15、满$150免运费
3. **交叉销售**：推荐搭配产品
4. **向上销售**：推荐高价位版本
5. **限时优惠**：加购专区、闪购

**监控维度**：
- 按渠道（不同流量来源的AOV）
- 按设备（移动端 vs 桌面端）
- 按时段（促销期 vs 常规期）
- 按客户类型（新客 vs 老客）

---

### 1.5 转化率（CVR - Conversion Rate）

**定义**：访客转化为客户的比例

**计算公式**：
```
CVR = (转化次数 / 访问次数) × 100%
```

**多层级转化率**：
```
1. 加购率 = (加购次数 / 访问次数) × 100%
2. 结账率 = (进入结账 / 加购次数) × 100%
3. 完成率 = (完成支付 / 进入结账) × 100%
4. 整体转化率 = (订单数 / 访问次数) × 100%
```

**电商转化率基准**：
| 品类 | 平均转化率 |
|------|-----------|
| 食品饮料 | 3.0% - 4.5% |
| 服装配饰 | 2.0% - 3.0% |
| 电子产品 | 1.0% - 2.0% |
| 家居装饰 | 2.5% - 3.5% |
| 健康美容 | 2.5% - 4.0% |

**转化率优化要点**：
- 网站加载速度（< 3秒）
- 移动端体验优化
- 清晰的产品信息
- 简化结账流程
- 信任元素（评价、认证、退换政策）

---

### 1.6 购物车放弃率（Cart Abandonment Rate）

**定义**：将商品加入购物车但未完成购买的用户比例

**计算公式**：
```
购物车放弃率 = (1 - 完成购买数 / 加购数) × 100%
```

**行业平均**：约 70%

**常见放弃原因及对策**：
| 原因 | 占比 | 优化策略 |
|------|------|----------|
| 额外费用太高 | 48% | 提前显示总价、免运费门槛 |
| 需要创建账户 | 26% | 提供游客结账 |
| 结账流程复杂 | 23% | 简化步骤、进度指示 |
| 网站不信任 | 18% | 安全徽章、评价展示 |
| 支付方式少 | 13% | 增加支付选项 |

---

### 1.7 复购率（Repeat Purchase Rate）

**定义**：有重复购买行为的客户占比

**计算公式**：
```
复购率 = (有2次及以上购买的客户数 / 总客户数) × 100%
```

**相关指标**：
- **新客占比**：首次购买客户比例
- **老客贡献率**：老客户收入占总收入比例
- **复购周期**：两次购买的平均间隔时间

**提升复购率的策略**：
1. **邮件营销**：生命周期邮件、弃单召回
2. **会员体系**：积分、等级、专属优惠
3. **订阅模式**：周期性购买产品
4. **社交媒体**：社群运营、内容营销
5. **客户体验**：优质服务、快速响应

---

### 1.8 广告点击率（CTR - Click-Through Rate）

**定义**：广告展示后被点击的比例

**计算公式**：
```
CTR = (点击次数 / 展示次数) × 100%
```

**不同广告形式的基准**：
| 广告类型 | 平均CTR |
|----------|---------|
| Google搜索广告 | 3% - 5% |
| Google展示广告 | 0.5% - 1% |
| Facebook广告 | 0.9% - 1.5% |
| 邮件营销 | 2% - 4% |

---

### 1.9 每次点击成本（CPC - Cost Per Click）

**定义**：每次广告点击的费用

**计算公式**：
```
CPC = 广告费用 / 点击次数
```

**优化方向**：
- 提高广告质量得分（Google Ads）
- 优化关键词精准度
- 改进广告创意
- 调整出价策略

---

### 1.10 千次展示成本（CPM - Cost Per Mille）

**定义**：广告每展示1000次的费用

**计算公式**：
```
CPM = (广告费用 / 展示次数) × 1000
```

**适用场景**：
- 品牌曝光目标
- 展示广告、社交媒体广告
- 评估不同渠道的曝光成本

---

### 1.11 核心指标关系图

```
流量指标 → 互动指标 → 转化指标 → 收入指标
  ↓           ↓           ↓           ↓
CTR/CPC    跳出率      CVR         AOV
CPM        停留时间    加购率       ROAS
           页面深度    放弃率       LTV
                                   CAC
```

**关键公式链**：
```
收入 = 流量 × CVR × AOV
利润 = 收入 - 成本
ROAS = 收入 / 广告成本
ROI = (收入 - 成本) / 成本
```

## 2. Google Analytics 4 配置指南

### 2.1 GA4 与 Universal Analytics 的主要区别

| 特性 | Universal Analytics | Google Analytics 4 |
|------|---------------------|-------------------|
| 数据模型 | 基于会话 | 基于事件 |
| 用户追踪 | 依赖Cookie | 多种ID（User ID、Google信号、Device ID） |
| 隐私保护 | 较弱 | 增强隐私控制、无需Cookie也能工作 |
| 机器学习 | 有限 | 内置预测功能 |
| 免费数据保留 | 不限制 | 2个月（可延长至14个月） |
| 跨平台追踪 | 需单独配置 | 原生支持 |

---

### 2.2 GA4 账户创建步骤

#### 步骤1：创建账户和媒体资源

1. 访问 [analytics.google.com](https://analytics.google.com)
2. 点击「开始测量」
3. 输入账户名称（公司名）
4. 创建媒体资源：
   - 媒体资源名称（网站名）
   - 报告时区（选择目标市场时区）
   - 货币（如USD、CNY）

#### 步骤2：添加数据流

**网站数据流配置**：
1. 选择平台：网站
2. 输入网站URL（如：https://yourstore.com）
3. 数据流名称
4. 增强型衡量功能：**开启**（自动追踪以下事件）
   - 页面浏览
   - 滚动
   - 出站点击
   - 站点搜索
   - 视频参与
   - 文件下载

#### 步骤3：安装追踪代码

**方式一：Google Tag（推荐）**
```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

**方式二：通过GTM（Google Tag Manager）**
1. 创建GTM容器
2. 添加GA4配置标签
3. 触发器：All Pages

**方式三：电商平台集成**
- Shopify：使用Google Channel
- WooCommerce：安装GA4插件
- Magento：后台配置

### 2.3 电商追踪配置（Enhanced Ecommerce）

GA4使用**事件追踪**记录电商行为：

#### 核心电商事件

| 事件名称 | 触发时机 | 必要参数 |
|----------|----------|----------|
| view_item | 查看产品页 | items, currency, value |
| select_item | 点击产品 | items |
| add_to_cart | 加入购物车 | items, currency, value |
| remove_from_cart | 移出购物车 | items, currency, value |
| view_cart | 查看购物车 | items, currency, value |
| begin_checkout | 开始结账 | items, currency, value |
| add_shipping_info | 添加配送信息 | items, currency, value, shipping_tier |
| add_payment_info | 添加支付信息 | items, currency, value, payment_type |
| purchase | 完成购买 | items, transaction_id, value, currency, tax, shipping |

#### 代码示例：购买事件

```javascript
gtag("event", "purchase", {
  transaction_id: "T12345",
  value: 125.50,
  currency: "USD",
  tax: 12.55,
  shipping: 5.00,
  items: [
    {
      item_id: "SKU12345",
      item_name: "Premium T-Shirt",
      item_category: "Apparel",
      item_variant: "Blue / L",
      price: 49.99,
      quantity: 2
    },
    {
      item_id: "SKU67890",
      item_name: "Canvas Hat",
      item_category: "Accessories",
      price: 25.51,
      quantity: 1
    }
  ]
});
```

#### items 数组必填参数

```javascript
{
  item_id: "SKU123",        // 产品SKU（与item_name二选一）
  item_name: "Product Name", // 产品名称
  item_brand: "Brand Name",  // 品牌（可选）
  item_category: "Category", // 类别（可选）
  item_variant: "Variant",   // 变体（可选）
  price: 29.99,              // 单价
  quantity: 1                // 数量
}
```

### 2.4 转化事件配置

ORDER BY event_date;

## 3. 转化漏斗分析和优化

### 3.1 电商转化漏斗模型

```
┌─────────────────────────────────────────┐
│           访问网站 (100%)                │  AWARENESS
└─────────────────────────────────────────┘
                    ↓ ~40-60%
┌─────────────────────────────────────────┐
│           浏览产品 (40-60%)              │  INTEREST
└─────────────────────────────────────────┘
                    ↓ ~20-30%
┌─────────────────────────────────────────┐
│           查看详情 (8-18%)               │  CONSIDERATION
└─────────────────────────────────────────┘
                    ↓ ~30-50%
┌─────────────────────────────────────────┐
│           加入购物车 (2.5-9%)            │  INTENT
└─────────────────────────────────────────┘
                    ↓ ~60-70%
┌─────────────────────────────────────────┐
│           开始结账 (0.75-2.7%)           │  EVALUATION

                    ↓ ~20-35%

**在GA4中创建漏斗**：
1. 探索 → 漏斗探索
2. 添加步骤（最多10步）
3. 分析：
   - 各步转化率
   - 放弃率
   - 细分对比

**示例漏斗设置**：
```

步骤8: purchase → 完成购买
```

#### 方法2：细分对比

**对比维度**：
- 设备类型：Mobile vs Desktop vs Tablet
- 流量来源：Organic vs Paid vs Social
- 新老客户：New vs Returning
- 地理位置：不同国家/地区
- 时段：工作日 vs 周末

**发现优化机会**：
- 移动端转化率低 → 优化移动体验
- 某渠道放弃率高 → 检查流量质量
- 老客转化率高 → 提升复购策略

#### 方法3：漏斗步骤流失分析

**分析每个步骤的流失用户行为**：
1. 流失用户去了哪里？
2. 是否返回上一步？
3. 是否离开网站？
4. 流失后是否再回来？

### 3.4 各阶段优化策略

#### 访问 → 浏览产品

**优化目标**：降低跳出率，引导深入浏览

**策略**：
- 优化落地页加载速度（< 3秒）
- 清晰的导航和分类
- 首页展示热门/新品产品
- 个性化推荐
- 吸引力的视觉设计

**诊断问题**：
- 跳出率 > 70%：落地页与广告不匹配
- 页面浏览 < 2：导航不清晰或内容不相关

#### 浏览产品 → 查看详情

**优化目标**：提高产品吸引力

**策略**：
- 优化产品缩略图
- 清晰的价格和促销信息
- 产品分类便于浏览
- 筛选和排序功能

#### 查看详情 → 加购

**优化目标**：提升加购率

**策略**：
1. **产品页面优化**
   - 高质量产品图片（多角度、细节）
   - 详细产品描述
   - 尺寸指南/规格表
   - 用户评价和UGC
   - 视频/3D展示

2. **价格策略**
   - 清晰的价格展示
   - 促销标签（折扣、限时）
   - 分期付款选项

3. **信任元素**
   - 退换货政策
   - 运送信息
   - 安全支付标志
   - 客户评价

4. **行动召唤**
   - 醒目的"加入购物车"按钮
   - 紧迫感（库存提示）
   - 一键加购

#### 加购 → 开始结账

**优化目标**：减少购物车放弃

**常见放弃原因及对策**：

| 原因 | 对策 |
|------|------|
| 额外费用高 | 提前显示总价、免运费门槛 |
| 还在比较 | 保存购物车、邮件提醒 |
| 价格敏感 | 限时优惠、优惠券 |
| 没准备好 | 愿望清单功能 |

**购物车页面优化**：
- 清晰的商品信息（图片、名称、价格、数量）
- 轻松修改数量
- 计算器功能（实时总价）
- 突出优惠信息
- 简洁的"去结账"按钮

#### 开始结账 → 完成购买

**优化目标**：简化结账流程，降低放弃率

**结账流程优化**：

1. **提供游客结账**
   - 不强制注册
   - 结账后可选注册

2. **简化表单**
   - 减少必填字段
   - 自动填充地址
   - 地址验证
   - 进度指示器

3. **多种支付方式**
   - 信用卡
   - PayPal
   - Apple Pay / Google Pay
   - 分期付款
   - 本地支付方式

4. **安全信任**
   - SSL证书
   - 安全徽章
   - 隐私政策链接
   - 客服联系信息

5. **移动端优化**
   - 大按钮
   - 数字键盘
   - 简化步骤

**结账页面不要做的**：
- 隐藏额外费用
- 强制创建账户
- 复杂的表单
- 有限支付选项
- 缺乏安全感

### 4.3 常见用户分群模型

#### RFM模型

**RFM定义**：
- **R（Recency）**：最近一次购买时间
- **F（Frequency）**：购买频率
- **M（Monetary）**：消费金额

**RFM评分方法**：
```
R评分（以1-5分为例）：
- 5分：30天内购买
- 4分：30-60天
- 3分：60-90天
- 2分：90-180天
- 1分：180天以上

F评分：
- 5分：10次以上
- 4分：6-9次
- 3分：3-5次
- 2分：2次
- 1分：1次

M评分（根据业务调整）：
- 5分：$500以上
- 4分：$200-499
- 3分：$100-199
- 2分：$50-99
- 1分：$50以下
```

**RFM用户分群**：

| 分群 | RFM特征 | 描述 | 营销策略 |
|------|---------|------|----------|
| 重要价值客户 | 555 | 最近购买、高频、高消费 | VIP服务、专属优惠、新品体验 |
| 重要发展客户 | 515/525 | 最近购买、低频、高消费 | 提升购买频率、忠诚度计划 |
| 重要保持客户 | 155/255 | 非近期、低频、高消费 | 召回活动、个性化推荐 |
| 重要挽留客户 | 355/455 | 较近期、高频、高消费 | 激活活动、专属客服 |
| 一般价值客户 | 353/453 | 中等水平 | 适度营销、提升转化 |
| 一般发展客户 | 513/523 | 最近购买、低频、低消费 | 培养习惯、交叉销售 |
| 一般保持客户 | 153/253 | 非近期、低频、低消费 | 低成本维护、再营销 |
| 流失客户 | 111/112 | 非近期、低频、低消费 | 放弃或低成本召回 |

**RFM分析示例**：

```
某服装独立站RFM分析：

重要保持客户（155/255）：3.8%
→ 年贡献收入：12%
→ 策略：召回邮件、限时优惠

流失风险客户：15.2%
→ 策略：再营销广告、问卷调研
```

## 5. A/B测试方法论

### 5.1 A/B测试基础概念

**定义**：A/B测试是一种对比两个版本（A版和B版）以确定哪个表现更好的实验方法。

**核心要素**：
- **控制组（A）**：原始版本
- **实验组（B）**：优化版本
- **变量**：唯一改变的元素
- **指标**：衡量成功的标准

---

### 5.2 A/B测试适用场景

#### 高优先级测试场景

| 优先级 | 测试元素 | 预期影响 |
|--------|----------|----------|
| 高 | 结账流程 | 转化率+5-15% |
| 高 | 产品页布局 | 加购率+10-20% |
| 高 | CTA按钮 | 点击率+10-30% |
| 中 | 首页设计 | 跳出率降低 |
| 中 | 导航结构 | 页面浏览增加 |
| 中 | 促销展示 | AOV提升 |
| 低 | 颜色/字体 | 微小影响 |
| 低 | 页脚设计 | 影响有限 |

---

### 5.3 A/B测试流程

#### 完整测试流程

```
┌─────────────────────────────────────────────────────┐
│ 1. 确定测试目标                                      │
│    - 明确要优化的指标                                │
│    - 设定预期提升目标                                │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ 2. 提出假设                                          │
│    - 基于数据分析发现问题                            │
│    - 提出"如果...那么..."假设                        │
└─────────────────────────────────────────────────────┘

### 5.4 样本量计算

#### 样本量影响因素

```
所需样本量取决于：
1. 当前转化率（基准率）
2. 预期提升幅度（MDE - 最小可检测效应）
3. 统计显著性水平（通常95%）
4. 统计功效（通常80%）
```

#### 样本量计算公式

```
样本量 = 16 × σ² / Δ²

其中：
σ² = p × (1-p) （方差）
p = 当前转化率
Δ = 预期提升差值
```

#### 样本量参考表

| 当前转化率 | 预期提升 | 每组所需样本 |
|------------|----------|--------------|
| 1% | +10% | 760,000 |
| 1% | +20% | 190,000 |
| 2% | +10% | 380,000 |
| 2% | +20% | 95,000 |
| 5% | +10% | 152,000 |
| 5% | +20% | 38,000 |

**经验法则**：
- 大流量网站：可检测小提升（5-10%）
- 中等流量：聚焦大提升（15-30%）
- 小流量：优先定性优化或大幅改版

### 5.5 统计显著性

#### 关键概念

- **P值**：结果由随机产生的概率
  - P < 0.05 = 95%置信度 = 统计显著
  
- **置信区间**：真实提升的范围
  - 例：提升10% ± 2%，即8%-12%

- **统计功效**：检测到真实差异的能力
  - 通常设定为80%

#### 判断标准

| P值范围 | 置信度 | 判定 |
|---------|--------|------|
| P < 0.01 | > 99% | 非常显著 |
| P < 0.05 | > 95% | 显著 |
| P < 0.10 | > 90% | 边缘显著 |
| P > 0.10 | < 90% | 不显著 |

### 5.6 A/B测试最佳实践

#### 测试设计原则

1. **一次只测一个变量**
   ```
   ✅ 正确：只改变CTA按钮颜色
   ❌ 错误：同时改变按钮颜色、文案、位置
   ```

2. **测试时间足够长**
   - 最短：7天（覆盖完整周周期）
   - 建议：2-4周
   - 考虑业务周期（工作日vs周末、发薪日等）

3. **流量分配均匀**
   - 50/50分配（标准）
   - 90/10分配（风险控制）

4. **避免提前结束**
   - 等达到预设样本量
   - 避免"偷看"效应

#### 常见测试元素

**1. CTA按钮测试**
```
测试变量：
- 颜色：红色 vs 绿色 vs 蓝色
- 文案："立即购买" vs "加入购物车" vs "获取优惠"
- 大小：标准 vs 加大
- 形状：圆角 vs 直角
- 位置：顶部 vs 中部 vs 底部
```

**2. 首页测试**
```
测试变量：
- Hero图片：产品图 vs 场景图 vs 视频
- 标题文案：功能导向 vs 情感导向
- 优惠展示：百分比折扣 vs 金额折扣
- 导航布局：横向 vs 汉堡菜单
```

**3. 产品页测试**
```
测试变量：
- 图片展示：轮播 vs 缩略图
- 评价位置：产品描述上方 vs 下方
- 库存提示：显示 vs 不显示
- 相关产品：横向展示 vs 纵向展示
```

**4. 结账流程测试**
```
测试变量：
- 步骤数量：单页 vs 多步
- 游客结账：显示 vs 隐藏
- 支付方式顺序：信用卡在前 vs PayPal在前
- 进度条：显示 vs 不显示
```

### 5.7 测试案例分析

#### 案例1：CTA按钮颜色测试

**假设**：将"加入购物车"按钮从灰色改为橙色可以提升加购率

**测试设置**：
- A版：灰色按钮
- B版：橙色按钮
- 流量分配：50/50
- 主要指标：加购率
- 样本量：每组50,000访客

**结果**：
```
A版（灰色）：加购率 4.2%
B版（橙色）：加购率 4.8%
提升：+14.3%
P值：0.012（显著）
置信区间：+8% 到 +21%
```

**结论**：橙色按钮获胜，全站部署

#### 案例2：结账流程简化测试

**假设**：将4步结账改为2步可以提高完成率

**测试设置**：
- A版：4步结账（配送→支付→确认→完成）
- B版：2步结账（配送+支付→确认）
- 主要指标：结账完成率

**结果**：
```
A版（4步）：完成率 68%
B版（2步）：完成率 72%
提升：+5.9%
```

**细分发现**：
- 桌面端：提升+2%
- 移动端：提升+11%
→ 移动端受益更大

## 6. 归因模型选择

### 6.1 什么是归因模型

**定义**：归因模型是确定多个营销触点如何贡献于转化的规则。

**为什么重要**：
- 了解哪些渠道真正有效
- 优化广告预算分配
- 评估营销组合ROI

---

### 6.2 常见归因模型

#### 1. 最后点击归因（Last Click）

**定义**：100%归因于转化前的最后一个触点

```
用户路径：Facebook广告 → Google搜索 → 直接访问 → 转化
归因：直接访问 100%

优点：简单明了
缺点：忽略早期触点贡献
适用：短决策周期、直接响应广告
```

#### 2. 首次点击归因（First Click）

**定义**：100%归因于第一个触点

```
用户路径：Facebook广告 → Google搜索 → 直接访问 → 转化
归因：Facebook广告 100%

优点：认可获客渠道价值
缺点：忽略后续触点
适用：品牌认知阶段评估
```

#### 3. 线性归因（Linear）

**定义**：所有触点平均分配归因

```
用户路径：Facebook广告 → Google搜索 → 直接访问 → 转化
归因：每个触点 33.3%

优点：认可所有触点贡献
缺点：可能高估低价值触点
适用：触点数量少、周期短
```

#### 4. 时间衰减归因（Time Decay）

**定义**：越接近转化的触点获得越多归因

```
用户路径（7天跨度）：
Day 1: Facebook广告 → 10%
Day 3: Google搜索 → 20%
Day 7: 直接访问 → 70%

优点：重视近期触点
缺点：可能低估早期触点
适用：长销售周期
```

#### 5. 位置归因（Position Based / U型）

**定义**：首次和末次触点各40%，中间触点分享20%

```
用户路径：Facebook广告 → Google搜索 → 邮件 → 直接访问 → 转化
归因：
- Facebook广告：40%（首次）
- Google搜索：10%
- 邮件：10%
- 直接访问：40%（末次）

优点：平衡首末触点
缺点：固定比例可能不适用所有情况
适用：大多数电商场景
```

#### 6. 数据驱动归因（Data-Driven）

**定义**：基于机器学习算法，根据实际数据动态分配归因

```
算法分析：
- 各触点的转化率
- 触点组合的影响
- 控制组对比

优点：最精确、考虑交互效应
缺点：需要足够数据量
适用：大流量、多触点
```

---

### 6.3 各模型对比

| 模型 | 优点 | 缺点 | 最佳适用场景 |
|------|------|------|--------------|
| 最后点击 | 简单、易于理解 | 忽略上游贡献 | 直接响应、短周期 |
| 首次点击 | 识别获客来源 | 忽略转化推动 | 品牌认知评估 |
| 线性 | 公平分配 | 无法区分价值 | 少触点场景 |
| 时间衰减 | 重视近期影响 | 可能低估早期 | 长销售周期 |
| 位置(U型) | 平衡首末 | 固定比例 | 一般电商 |
| 数据驱动 | 最精确 | 需大数据 | 高流量站点 |

#### 方法2：网站行为数据分析

**高意向未购买产品分析**：
```sql
-- GA4/BigQuery分析示例
SELECT 
  item_id,
  item_name,
  COUNT(DISTINCT CASE WHEN event_name='view_item' THEN user_id END) as viewers,
  COUNT(DISTINCT CASE WHEN event_name='add_to_cart' THEN user_id END) as cart_adders,
  COUNT(DISTINCT CASE WHEN event_name='purchase' THEN user_id END) as buyers,
  SAFE_DIVIDE(
    COUNT(DISTINCT CASE WHEN event_name='add_to_cart' THEN user_id END),
    COUNT(DISTINCT CASE WHEN event_name='view_item' THEN user_id END)
  ) as add_to_cart_rate
FROM ecommerce_events
GROUP BY item_id, item_name
HAVING viewers > 1000
ORDER BY add_to_cart_rate DESC
```

**识别机会产品**：
- 高浏览低转化：优化产品页
- 高加购低购买：检查价格/竞品
- 高搜索无产品：开发新品

**竞品监控维度**：
- 新品上架频率
- 价格变化
- 评价数量和评分

### 8.8 看板设计最佳实践

#### 设计原则

1. **5秒法则**：关键信息5秒内可见
2. **3层结构**：概览 → 细分 → 原始数据
3. **对比展示**：同比、环比、目标对比
4. **颜色编码**：
   - 🟢 绿色：达标/优于预期
   - 🟡 黄色：接近目标
   - 🔴 红色：低于目标/异常

#### 常见错误

| 错误 | 问题 | 改进 |
|------|------|------|
| 指标过多 | 信息过载 | 精简到10-15个核心指标 |
| 缺乏对比 | 无法判断好坏 | 添加同比、环比、目标 |
| 更新不及时 | 数据过时 | 自动化数据更新 |
| 无行动指引 | 看了不知道做什么 | 添加预警和建议 |
| 不分受众 | 所有人看同一看板 | 分层设计不同看板 |

## 总结：独立站数据分析体系框架

### 核心指标速查表

| 类别 | 指标 | 公式 | 健康基准 |
|------|------|------|----------|
| **获客** | CAC | 营销费用/新客户数 | < LTV/3 |
| **获客** | CTR | 点击/展示 | > 1% |
| **获客** | CPC | 费用/点击 | 因渠道而异 |
| **转化** | CVR | 订单/访问 | 1.5-3% |
| **转化** | 加购率 | 加购/访问 | 5-10% |
| **转化** | 放弃率 | 1-完成/加购 | < 70% |
| **收入** | AOV | 收入/订单 | 因品类而异 |
| **收入** | ROAS | 收入/广告费 | > 300% |
| **用户** | LTV | AOV×频率×周期×毛利 | > 3×CAC |
| **用户** | 复购率 | 复购客户/总客户 | > 20% |
| **用户** | 留存率 | 回访用户/总用户 | 7日>15% |
