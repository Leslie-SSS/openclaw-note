# 382. 支付合规体系

## 核心要点

### 1. 支付合规的核心领域

**牌照合规**：
- 支付业务许可（MSB、PI等）
- 各国支付牌照
- 跨境支付资质
- 数据处理许可

**反洗钱（AML）**：
- 客户身份识别（KYC）
- 交易监控
- 可疑交易报告
- 制裁名单筛查

**数据合规**：
- 数据保护法（GDPR、CCPA等）
- 数据本地化要求
- 支付数据安全标准（PCI DSS）
- 跨境数据传输

**消费者保护**：
- 透明披露
- 退款政策
- 纠纷处理
- 隐私保护

## 实操建议

### 1. 合规策略选择

**策略A：自主合规**（大型企业）
```
适用：
- 年GMV > $100M
- 有专业合规团队
- 长期战略市场

优势：
- 完全控制
- 品牌信誉

挑战：
- 成本高
- 周期长
```

**策略B：借助服务商**（中小型企业）
```
适用：
- 年GMV < $100M
- 多市场运营
- 快速扩张

优势：
- 成本低
- 快速上线
- 专业支持

方式：
- 使用持牌支付服务商
- White-label方案
- 合规托管
```

**策略C：混合策略**（中型企业）
```
核心市场：自主合规
次要市场：服务商
新兴市场：测试+评估
```

### 2. KYC/AML实施

**KYC流程**：
```
个人客户：
1. 基础信息
   - 姓名、出生日期
   - 地址
   - 联系方式

2. 身份验证
   - 护照/身份证
   - 地址证明
   - 人脸识别

3. 风险评估
   - PEP筛查
   - 制裁名单
   - 风险等级

企业客户：
1. 公司信息
   - 注册文件
   - 营业执照
   - 公司章程

2. 实益所有人
   - 25%+股东信息
   - 董事信息
   - UBO声明

3. 业务验证
   - 业务模式
   - 资金来源
   - 财务报表
```

**分级KYC**：
```
┌──────────┬──────────────┬────────────┐
│ 交易金额  │ KYC要求       │ 验证方式    │
├──────────┼──────────────┼────────────┤
│ < $100   │ 基础信息      │ 邮箱验证    │
│ $100-1K  │ 标准KYC       │ ID验证     │
│ $1K-10K  │ 增强KYC       │ 地址证明    │
│ > $10K   │ EDD           │ 完整尽调    │
└──────────┴──────────────┴────────────┘
```

### 3. 交易监控系统

**监控规则**：
```javascript
// 交易监控规则示例
const monitoringRules = [
    {
        rule: 'large_amount',
        condition: 'amount > 10000',
        action: 'flag_for_review'
    },
    {
        rule: 'velocity_check',
        condition: 'transaction_count_24h > 10',
        action: 'block_and_verify'
    },
    {
        rule: 'country_risk',
        condition: 'country in HIGH_RISK_COUNTRIES',
        action: 'enhanced_due_diligence'
    },
    {
        rule: 'sanction_match',
        condition: 'name_match_sanction_list',
        action: 'block_immediately'
    }
];

function monitorTransaction(transaction) {
    for (const rule of monitoringRules) {
        if (evaluateCondition(rule.condition, transaction)) {
            executeAction(rule.action, transaction);
            logAlert(rule.rule, transaction);
        }
    }
}
```

**监控指标**：
```
实时监控：
- 单笔金额异常
- 频率异常
- 地理位置异常
- 设备指纹异常

定期监控：
- 交易模式分析
- 客户行为分析
- 商户风险评分
```

### 4. PCI DSS合规

**12项核心要求**：
```
1. 安装防火墙
2. 更改默认密码
3. 保护存储数据
4. 加密传输数据
5. 使用防病毒
6. 更新系统补丁
7. 限制访问权限
8. 唯一ID标识
9. 物理访问控制
10. 访问日志
11. 定期安全测试
12. 信息安全政策
```

**合规等级**：
```
SAQ A：完全外包
- 使用PSP/网关
- 不接触卡数据
- 最简单

SAQ A-EP：部分外包
- 部分处理卡数据
- 中等复杂度

SAQ D：自行处理
- 完全自主
- 最复杂
```

**推荐做法**：
- 使用PCI认证的支付服务商
- 不存储CVV/CVC
- 使用令牌化（Tokenization）
- 定期安全审计

### 5. 数据保护合规

**GDPR要点**：
```
合法性基础：
- 用户同意
- 合同履行
- 合法利益

数据主体权利：
- 访问权
- 更正权
- 删除权
- 携带权
- 反对权

合规措施：
✓ 隐私政策
✓ 同意管理
✓ 数据映射
✓ DPO任命（如需要）
✓ 数据泄露通知
```

**数据本地化**：
```
有本地化要求的国家：
- 中国（数据安全法）
- 俄罗斯
- 印度（部分）
- 越南
- 印尼（部分）

应对策略：
- 使用本地数据中心
- 与本地服务商合作
- 数据分类存储
```

## 关键案例

### 案例：Stripe的全球合规体系

**背景**：
Stripe在全球46+国家运营，支持135+货币，需要应对复杂的合规环境。

**合规架构**：

1. **全球合规团队**
   - CCO领导
   - 100+合规人员
   - 分布在全球办公室

2. **牌照策略**
   - 美国MSB + 各州许可
   - 欧盟EMI牌照（卢森堡）
   - 英国FCA授权
   - 亚太各地牌照

3. **技术驱动的合规**
   - 自动化KYC
   - AI交易监控
   - 实时制裁筛查

4. **合规即产品**
   - Stripe Radar（反欺诈）
   - Stripe Identity（身份验证）
   - Stripe Tax（税务）

**成果**：
- 99.99%合规准确率
- 处理时间< 2分钟
- 欺诈损失< 0.1%
- 监管零处罚

**关键经验**：
- 技术投入带来效率
- 合规是产品，不是成本
- 全球视野+本地执行

### 案例：中小企业的合规路径

**背景**：
某跨境电商，年GMV $20M，业务覆盖美国、欧洲、东南亚。

**合规方案**：

1. **借助服务商**
   - 使用Airwallex（持牌）
   - 通过Stripe/Adyen收单
   - 不自持牌照

2. **内部合规**
   - 合规负责人1人
   - 外包KYC验证
   - 使用ComplyAdvantage监控

3. **成本控制**
   - 年合规成本：$50K
   - 占收入：0.25%
   - 性价比最优

**成果**：
- 快速扩张到15国
- 合规零问题
- 成本可控

## 总结

支付合规体系的核心是**风险控制、成本平衡、业务支持**。通过选择合适的合规策略（自主、外包、混合）、实施有效的KYC/AML流程、确保数据合规、建立专业团队，企业可以在合规的前提下高效运营。关键在于根据规模选择策略，技术驱动效率，持续投入。

**合规清单**：
```
✓ 牌照：使用持牌服务商或自获牌照
✓ KYC/AML：分级验证、持续监控
✓ PCI DSS：不存储敏感数据、使用认证服务商
✓ 数据：GDPR合规、本地化要求
✓ 团队：专业团队+外部支持
✓ 审计：定期审计、及时报告
```
